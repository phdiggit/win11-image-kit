# 结果模型与报告验收

本文记录 #6 当前阶段用于收口的报告语义。它只描述 dry-run、fake report 和 CI 验收基线，不代表 #6 已关闭；阶段性 PR 统一使用 `Refs #6`。

## StepResult 状态语义

`StepResult` 是编排步骤的统一结果模型。常用状态如下：

- `changed`：真实执行后产生变更。
- `unchanged`：真实检查或执行后确认无需变更。
- `skipped`：范围、manifest 或参数关闭了步骤。
- `manual`：需要人工处理，不伪装为自动成功。
- `whatif`：`-WhatIf` 预演，必须保持 `changed = false`。
- `failed`：步骤失败；required 步骤失败计入阻断。

`stepSummary` 汇总编排步骤维度，包含状态计数、`failedRequiredCount`、`failedOptionalCount`、`hasBlockingFailure` 和建议 `exitCode`。

## Child report compact reference

顶层 build/postdeploy 报告只保存 compact child reference：路径、存在性、required 标记、错误码和各子报告 summary。顶层报告不得内嵌完整 `packageResults`、`serviceResults`、`junctionResults`、`defenderResults`、`appxResults` 或 `userExperienceResults` 数组；完整明细只保留在对应子报告文件里。

## Required / Optional failure

`childReportSummary` 是顶层报告对子报告维度的统一阻断汇总：

- required 子报告缺失、解析失败、summary 缺失或自身有 required failure 时，计入 `failedRequired`，并让 `hasBlockingFailure = true`、`exitCode = 1`。
- optional 子报告失败只计入 `failedOptional`，默认不阻断，`exitCode` 仍为 `0`。
- fake child report fixture 用于验证 required failure、optional failure、missing report 和 parse failed 的映射，不依赖真实系统状态。

## Top-level childReportSummary

build 顶层报告至少汇总 `packageReports`。postdeploy 顶层报告汇总已接入的 `packageReports`、`serviceReports`、`junctionReports`、`defenderReports` 和 `userExperienceReports`；`byType.appx` 可以保持空计数，直到 postdeploy 主链路接入 AppX 子报告。

Markdown 顶层报告也应输出子报告总数、缺失数、`failedRequired`、`failedOptional`、`hasBlockingFailure` 和建议 `exitCode`，用于人工验收时快速判断是否存在假成功。

## PR_READY 与 Fast / Full CI

PR_READY 交付时，本地必要验证必须完成，PR 保持 ready for review。PR Fast CI 只跑快速阻断项和目标 Pester，适合覆盖 `StepResult`、compact child reference、`childReportSummary` 和 Markdown 汇总行；不得在 PR Fast CI 中新增真实 installer、服务、Junction、Defender、AppX、注册表、Sysprep、DISM、diskpart、WinPE 或 NAS 写入。

`Full Validate` 保留在 `main` push 或 `workflow_dispatch`，继续承担全量 Windows PowerShell 与 PowerShell 7 验证。除任务卡明确要求外，阶段性 PR 不等待 Full Validate。

## #6 收口前验收清单

- build `-WhatIf` 顶层 JSON 报告包含 `stepResults`、`stepSummary`、`packageReports` 和 `childReportSummary`。
- postdeploy `-WhatIf` 顶层 JSON 报告包含 `stepResults`、`stepSummary`、多类 child reports 和 `childReportSummary`。
- child references 保持 compact，不嵌入完整 `*Results` 数组。
- required failure、missing report 和 parse failed 会阻断。
- optional failure 不阻断。
- WhatIf 结果不伪装成 `changed` 或 `completed`。
- PR body 使用 `Refs #6`，不自动关闭 #6。
