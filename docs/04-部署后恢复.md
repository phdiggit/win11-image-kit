# 部署后恢复

目标：新机还原后，除账号登录和授权激活外，其余配置尽量自动完成。

## 执行时机

OOBE 完成、进入桌面、网络可用、Windows 初始驱动安装稳定后执行。

```powershell
Set-ExecutionPolicy Bypass -Scope Process -Force
.\scripts\postdeploy\Invoke-PostDeploy.ps1
```

如需先预演会执行哪些危险动作，使用：

```powershell
.\scripts\postdeploy\Invoke-PostDeploy.ps1 -WhatIf
```

## 自动恢复内容

- 按 `manifests/paths.json` 解析项目路径、工具目录和数据目录
- 按 `manifests/customization-scope.json` 确认本轮定制范围
- Windows Defender 排除项
- D 盘数据重定向
- 默认应用关联
- Windows Terminal 配置
- VSCode 便携配置检查
- 开始菜单固定项
- 数据库/中间件服务注册：执行 `manifests/services.json` 中的 `install` 命令，已存在服务会跳过，不自动启动服务
- 常用工具 PATH 验证
- 部署日志写入 NAS

## 人工清单

- 登录 Chrome
- 登录 VSCode Settings Sync
- 登录 JetBrains 账号或重新激活
- 登录微信、QQ、Telegram、网盘
- Tailscale 重新认证
- 需要时安装 Visual Studio、VMware Workstation、Adobe 系列

## 验证

先运行只检查声明和脚本结构的轻量测试：

```powershell
.\scripts\tests\Test-PostDeploy.ps1 -SkipCommandTests -SkipServiceStatus
```

确认新机上的软件和服务已经安装后，再运行完整测试：

```powershell
.\scripts\tests\Test-PostDeploy.ps1
```
